বাংলা গাইড: `.env.example` key কোথায় পাবেন, কীভাবে set করবেন, এবং কোন secret কী কাজ করে

এই ডকুমেন্টটি client/developer handover-এর জন্য। এখানে real secret value রাখা হয়নি। প্রতিটি key কোথা থেকে নিতে হবে, project-এ কীভাবে বসাতে হবে, এবং কোনটা public/secret তা ব্যাখ্যা করা হয়েছে।

১. সবচেয়ে জরুরি নিয়ম

Secret কখনো GitHub, frontend code, screenshot, chat, বা public document-এ দিবেন না।

NEXT_PUBLIC_ দিয়ে শুরু হওয়া key browser-এ দেখা যেতে পারে। বাকি secret key backend-only ধরে রাখতে হবে। কোনো secret ভুলে share হয়ে গেলে provider dashboard থেকে key rotate/delete করে নতুন key বসাতে হবে।

  • Local development: project root-এ .env.local file-এ value বসাবেন।
  • Production/Vercel: Vercel Dashboard → Project → Settings → Environment Variables-এ value বসাবেন, তারপর redeploy করবেন।
  • এই project-এর dev server সাধারণত http://localhost:3002 এ চলে, তাই local URL key-তে 3002 ব্যবহার করা ভালো।
  • .env / .env.local GitHub-এ push করা যাবে না। এই project-এর .gitignore already env file ignore করে।

২. Local setup কীভাবে করবেন

  1. Project root খুলুন: E:\programming hero\Best Bazar
  2. .env.example copy করে .env.local বানান।
  3. প্রতিটি KEY=VALUE line-এ আপনার provider dashboard থেকে পাওয়া value বসান।
  4. Server restart করুন: npm run dev
Copy-Item .env.example .env.local
notepad .env.local
npm run dev
Local URL note:

`.env.example`-এ যদি http://localhost:3000 থাকে, এই project চালানোর সময় প্রয়োজন হলে http://localhost:3002 করুন, কারণ custom dev script port 3002 ব্যবহার করে।

৩. Production/Vercel setup

  1. Vercel Dashboard-এ login করুন।
  2. Best Bazar project খুলুন।
  3. Settings → Environment Variables-এ যান।
  4. প্রতিটি key আলাদা করে add করুন। Environment হিসেবে Production, Preview, Development প্রয়োজন অনুযায়ী select করুন।
  5. সব key save করার পর নতুন deployment দিন বা Redeploy করুন।
Vercel environment variables source code-এর বাইরে থাকে এবং deployment runtime-এ app এগুলো পড়ে। তাই production secret রাখার জন্য Vercel env/admin secrets সবচেয়ে clean জায়গা।

৫. Database / Neon keys

DATABASE_URLSECRET
কোথায় পাবেন
Neon Console → Project Dashboard → Connect → Connection pooling toggle ON → pooled connection string copy করুন।
কাজ
App-এর Prisma ORM এই URL দিয়ে PostgreSQL database connect করে। Product, order, user, settings সব data এখানে থাকে।
Format
DATABASE_URL=postgresql://USER:PASSWORD@HOST-pooler.REGION.aws.neon.tech/DB?sslmode=require&channel_binding=require&connection_limit=5&pool_timeout=20&connect_timeout=10

Production/serverless app-এর জন্য pooled hostname-এ -pooler থাকা ভালো। Migration বা manual DB admin task করলে direct non-pooled URL লাগতে পারে।

DATABASE_CONNECTION_LIMITCONFIG
কোথায় পাবেন
এটা provider থেকে copy করা key না; project tuning value। Default: 5
কাজ
Prisma/Neon pooled connection একসাথে কত connection ব্যবহার করবে সেটা control করে।
DATABASE_POOL_TIMEOUT / DATABASE_CONNECT_TIMEOUTCONFIG
কোথায় পাবেন
Manual value; সাধারণত example-এর 20 এবং 10 ঠিক আছে।
কাজ
DB connection wait এবং initial connect timeout control করে। Neon slow/temporary sleep হলে এগুলো help করে।

১৪. Secret leak হলে কী করবেন

Stripe

Dashboard → Developers → API keys → exposed key rotate/delete করুন। Webhook secret leak হলে endpoint secret rotate করুন।

Neon

Role/password rotate করুন অথবা নতুন database role বানিয়ে নতুন pooled connection string বসান।

Cloudinary

Settings → API Keys → Generate New Access Key, app update, তারপর old key disable করুন।

Google / Twilio / Email

Client secret/Auth token/App password rotate করুন এবং Vercel/local env update করে redeploy/restart দিন।
